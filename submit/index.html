模板申报

通过 GitHub Issue 表单提交模板上架申请。填写模板仓库、使用命令、功能简介和演示信息，方便维护者快速审核并收录。

打开 GitHub 申报表单 查看制作指南

提交前请准备

这个页面只负责引导，真正的申报会在 GitHub Issue 表单里完成，建议你先准备好以下信息。

  • 模板名称、仓库地址，以及一句话介绍
  • 模板分类、标签、初始化命令和适用场景
  • 可选的在线演示、截图、文档链接
  • 维护状态、依赖版本、已知限制等补充说明

提交后会发生什么

Issue 创建后，维护者会直接在 GitHub 中跟进审核流程，沟通记录也会完整保留。

  1. 先确认模板仓库、说明文档和命令是否可用
  2. 如有缺失信息，会在 Issue 中继续补充沟通
  3. 审核通过后，再安排收录到模板市场
  4. 后续更新也可以继续在同一个 Issue 中追踪

使用 GitHub 原生表单提交

当前站点是静态站点，页面只提供入口。实际申报采用 GitHub Issue Form，便于流程公开、可追踪、可协作。
