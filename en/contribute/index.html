Contribution Guide

Contribute your template to the Jzero Template Market

什么是第三方模板？

第三方模板是由社区开发者创建并维护的项目模板，可以帮助其他开发者快速启动特定类型的项目。

模板结构

每个第三方模板需要包含以下文件：

your-template/
├── template.json          # 模板配置文件（必需）
├── template.zh.json       # 中文配置文件（可选）
├── README.md              # 模板说明文档（必需）
└── [template files]       # 模板文件和目录

配置文件说明

模板配置文件是 JSON 格式，包含以下字段：

{
  "id": "your-template-id",
  "name": "Your Template Name",
  "description": "A brief description of your template",
  "category": "Category Name",
  "tags": ["tag1", "tag2", "tag3"],
  "command": "jzero new project_name --template your-template-id",
  "repository": "https://github.com/username/repo",
  "demo": "https://your-demo-url.com",
  "features": [
    "Feature 1",
    "Feature 2",
    "Feature 3"
  ]
}

字段说明

字段 类型 必需 说明
id string ✅ 模板唯一标识符
name string ✅ 模板名称
description string ✅ 模板简短描述
category string ✅ 模板分类
tags array ✅ 模板标签数组
command string ✅ 使用模板的命令
repository string ❌ 模板源码仓库地址
demo string ❌ 在线演示地址
features array ❌ 模板特性列表

多语言支持

如果需要支持中文，创建 template.zh.json 文件：

{
  "id": "your-template-id",
  "name": "模板中文名称",
  "description": "模板的中文描述",
  "category": "分类名称",
  "tags": ["标签1", "标签2"],
  "command": "jzero new project_name --template your-template-id",
  "repository": "https://github.com/username/repo",
  "demo": "https://your-demo-url.com",
  "features": [
    "特性1",
    "特性2"
  ]
}

最佳实践

  • 模板命名: 使用有意义的名称，推荐格式：用途-技术栈-特性
  • 分类选择: Web、API、CLI、Mobile、Desktop 等
  • 标签使用: 添加 3-8 个相关标签，包含技术栈和项目类型
  • 文档编写: README.md 应包含项目简介、功能特性、快速开始指南

提交模板

  1. 准备提交：确保包含完整的 template.json 和 README.md
  2. Fork 本仓库
  3. 在 third_party/ 目录下创建您的模板
  4. 创建 Pull Request
  5. 等待审核
