Prepare Before You Submit

The issue form is meant for template creators who want maintainers to help review and list a template in the marketplace.

  • Template name, repository URL, and a short project description
  • Applicable category, tags, and the command used to create the project
  • Optional online demo, screenshots, or documentation links
  • Notes about maturity, maintenance status, and supported language versions

What Happens Next

After the issue is submitted, maintainers can review the information directly in GitHub and follow up in the same thread.

  1. Maintainers confirm the template is complete and accessible
  2. They may ask for missing metadata, docs, or demo details
  3. Once approved, the template can be added to the public listing
  4. Follow-up updates can continue in the same GitHub issue

Prefer GitHub-native Submission

The static page only acts as a launcher. The actual submission happens in GitHub Issue forms so the review process stays transparent and traceable.
